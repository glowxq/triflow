## 1. 项目概述

### 1.1 项目定位

**Triflow**（Tri = 三端，Flow = 流动）是一个面向三端的企业级全栈快速开发脚手架：

| 端         | 仓库               | 技术栈                       | 说明                    |
|-----------|------------------|---------------------------|-----------------------|
| **后端**    | `triflow-server` | Java 21 + Spring Boot 3.5 | API 服务、业务逻辑           |
| **Web 端** | `triflow-web`    | Vue 3 + Vite              | 管理后台、官网、H5 等          |
| **移动端**   | `triflow-app`    | UniApp (Vue 3)            | 微信小程序 / Android / iOS |

帮助开发者快速构建高质量的企业级应用，让数据在各端之间流畅互通。

### 1.2 目标用户

- 中小型企业开发团队
- 独立开发者
- 需要快速搭建全栈应用的项目

### 1.3 设计原则

| 原则       | 说明                  |
|----------|---------------------|
| **模块化**  | 功能按模块划分，可按需引入，降低耦合  |
| **开箱即用** | 集成常用功能，减少重复开发和配置工作  |
| **易于扩展** | 提供清晰的扩展点，方便定制化开发    |
| **安全优先** | 内置安全防护机制，防止常见安全漏洞   |
| **规范统一** | 统一的代码风格、API 规范、异常处理 |

|-----|------|---------|:------------------------|
| T   | 错误类型 | 1-3     | 1:业务异常, 2:告警异常, 3:客户端异常 |
| MM  | 模块编码 | 00-99   | 参见模块编码表                 |
| CCC | 错误序号 | 000-999 | 模块内错误序号                 |

**错误类型说明：**

| 类型 | 前缀范围          | 异常类               | 说明               |
|----|---------------|-------------------|:-----------------|
| 1  | 100000-199999 | BusinessException | 业务异常，仅记录日志       |
| 2  | 200000-299999 | AlertsException   | 告警异常，记录日志并发送告警通知 |
| 3  | 300000-399999 | ClientException   | 客户端异常，前端需特殊处理    |

**模块编码表：**

| 编码  | 模块    | 说明       |
|-----|-------|:---------|
| 00  | 通用/系统 | 通用系统错误   |
| 01  | 认证授权  | 登录、权限相关  |
| 02  | 参数校验  | 参数验证错误   |
| 03  | 数据库   | 数据库操作错误  |
| 04  | 缓存    | Redis相关  |
| 05  | 文件存储  | OSS相关    |
| 07  | Excel | 导入导出错误   |
| 08  | 短信    | SMS相关    |
| 09  | 微信    | 微信相关     |
| 10  | 外部服务  | 第三方服务调用  |
| 20+ | 系统管理  | 用户/角色/菜单 |
| 50+ | 业务模块  | 各业务自定义   |

**特殊错误码：**

| 错误码 | 说明   |
|-----|:-----|
| 0   | 操作成功 |

### 10.2 API 规范

- RESTful 风格（不使用路径参数）
- 统一前缀 `/api/v1`
- 统一使用 `ApiResult` 返回数据
- 分页数据通过 `PageResult` 自动解析

**普通响应格式：**

```json
{
  "code": 0,
  "message": "SUCCESS",
  "data": {...},
  "requestId": "abc123"
}
```

**分页响应格式（自动解析 PageResult）：**

```json
{
  "code": 0,
  "message": "SUCCESS",
  "data": [...],
  "current": 1,
  "limit": 10,
  "totalPage": 5,
  "total": 50,
  "requestId": "abc123"
}
```
